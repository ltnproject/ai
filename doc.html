01 — Getting Started

🚀 Quick Start

DrkChat is ready to use out of the box — no downloads, no installs. Just open the app and start chatting. Your first 15 messages are completely free every day, no account required.

1
Open the App
Head to ltnproject.qzz.io — the app loads instantly in your browser. No installation needed.
2
Sign In or Try it Free
Sign in with Google, GitHub, or email to sync your chat history. Or skip — you get 15 free messages per day without an account.
3
Pick a Mode & Model
Choose a chat mode from the bottom bar (Chat, Code, Translate, etc.) and pick your AI model from the sidebar.
4
Start Chatting
Type your message and hit Enter to send. Use Shift+Enter for a new line. Press ↑ to recall your last message.
💡
Pro tip: Once your 15 free messages run out, grab a free Groq API key at console.groq.com — it's completely free and takes under a minute. Paste it in the sidebar to get unlimited messages.
02 — Authentication

🔐 Signing In

DrkChat uses Firebase Authentication to keep your chat history synced across all your devices. You have three ways to sign in:

🔵
Google
One click — uses your existing Google account. Fastest way to get started. Your profile photo appears in the chat.
⚫
GitHub
Perfect for developers. Authenticate with your GitHub account in one click.
📧
Email & Password
Create an account with your email. Use "Forgot password?" to reset it anytime via email link.
👤
Guest Mode
Skip sign-in entirely. Get 15 free messages per day. History isn't saved across sessions.
✅
Privacy: Your chat history is private — only you can see it. Each user's sessions are stored separately in Firestore with security rules that prevent anyone else from accessing your data.

Takes your text and rewrites it to be more clear, effective, and engaging.

05 — History & Settings

📚 Chat History & Settings

Chat History

When signed in, all your conversations are automatically saved to Firebase Firestore and synced across devices. Your history appears in the left sidebar, sorted by most recent.

Click any session to load it. Hover and click ✕ to delete it permanently. Click ✦ New Chat to start fresh.

Settings Panel

Click the ⚙️ button in the top bar to open Settings:

⚙️
Temperature — controls creativity. 0 = focused, 2 = very random
Max Tokens — maximum response length (512 to 8192)
Translate To — default target language for translate mode
Response Style — Balanced, Concise, Detailed, Academic, Casual, Technical
Streaming — toggle real-time token streaming on/off

System Prompt / Persona

Click 🧠 to set a custom system prompt or pick a preset persona (Expert, Coder, Tutor, Dark Writer, Analyst, Critic, Translator). The system prompt shapes how the AI behaves for your entire session.

06 — API Keys

🔑 Using Your Own API Key

DrkChat gives you 15 free messages per day using a shared Groq key via Cloudflare Worker proxy. Once you hit the limit, you need your own key.

1
Go to console.groq.com
Create a free Groq account. It takes under 60 seconds.
2
Create an API Key
In the Groq Console, go to API Keys → Create API Key. Copy the key — it starts with gsk_.
3
Paste it in DrkChat
In the sidebar at the bottom, paste your key into the Your API Key field. It's saved in your browser — never sent to any server except Groq directly.
🔒
Your key is secure. When you use your own key, requests go directly from your browser to Groq — bypassing the proxy entirely. The key is stored in localStorage and never leaves your device.
07 — Self Hosting

🛠️ Self-Hosting Overview

Want to run your own DrkChat instance? You need three things: a Firebase project (auth + database), a Cloudflare Worker (API proxy), and somewhere to host the HTML file.

🔥
Firebase
Authentication (Google, GitHub, Email) + Firestore for chat history storage. Free tier is very generous.
☁️
Cloudflare Worker
Proxies API requests to Groq so your API key stays secret. Free tier handles 100k requests/day.
⚡
Groq API
The actual AI engine. Create a free account at console.groq.com and get an API key.
🌐
Any Hosting
The app is a single HTML file. Host it anywhere — GitHub Pages, Netlify, your own server, Firebase Hosting.
08 — Firebase Setup

🔥 Firebase Setup

Create a Project

1
Go to console.firebase.google.com
Click Add Project, give it a name, and follow the setup wizard.
2
Enable Authentication
Go to Authentication → Sign-in method and enable Google, GitHub, and Email/Password. For GitHub you'll need a GitHub OAuth App.
3
Add Your Domain
Go to Authentication → Settings → Authorized domains and add your custom domain (e.g. yourdomain.com). Do NOT include https://.
4
Create Firestore Database
Go to Firestore Database → Create database. Start in production mode. Set these security rules:
firestore.rules rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /drkchat_sessions/{sessionId} {
      // Only owner can read/write/delete
      allow read, write, delete: if request.auth != null
        && request.auth.uid == resource.data.ownerId;
      // Only allow creating with your own ownerId
      allow create: if request.auth != null
        && request.resource.data.ownerId == request.auth.uid;
    }
  }
}

Update authDomain for Custom Domains

If you host on a custom domain, set authDomain to your domain in the Firebase config inside the HTML file:

drkchat.html const FB = initializeApp({
  apiKey: "your-api-key",
  authDomain: "yourdomain.com", // ← your custom domain
  projectId: "your-project-id",
  // ... rest of config
});
09 — Cloudflare Worker

☁️ Cloudflare Worker

The Worker acts as a secure proxy between your app and the Groq API — hiding your API key from users while enabling the shared 15-message free tier.

1
Go to workers.cloudflare.com
Create a free account, click Create Application → Create Worker.
2
Paste the Worker Code
Replace the default code with the worker script below and click Deploy.
3
Add Secret
Go to Settings → Variables → Secrets and add GROQ_API_KEY with your Groq key.
4
Update PROXY_URL in HTML
Copy your Worker URL (e.g. https://my-worker.name.workers.dev) and paste it as PROXY_URL in the HTML.
worker.js export default {
  async fetch(request, env) {
    const cors = {
      'Access-Control-Allow-Origin': '*',
      'Access-Control-Allow-Methods': 'POST, OPTIONS',
      'Access-Control-Allow-Headers': 'Content-Type',
    };
    if (request.method === 'OPTIONS')
      return new Response(null, { headers: cors });
    if (request.method !== 'POST')
      return new Response('Not allowed', { status: 405 });
    const body = await request.json();
    const res = await fetch(
      'https://api.groq.com/openai/v1/chat/completions',
      { method: 'POST', headers: { 'Authorization': `Bearer ${env.GROQ_API_KEY}`, 'Content-Type': 'application/json' }, body: JSON.stringify(body) }
    );
    const data = await res.text();
    return new Response(data, { status: res.status, headers: { ...cors, 'Content-Type': 'application/json' } });
  },
};
10 — Deploy

🌐 Deploying

DrkChat is a single HTML file — just drop it anywhere that serves static files. Here are the most popular options:

🔥
Firebase Hosting
Run firebase deploy after setting up Firebase CLI. Custom domain support built in.
🌿
Netlify
Drag and drop your HTML file at netlify.com/drop. Instant deploy with a free subdomain.
▲
Vercel
Push to GitHub and connect to Vercel for automatic deploys on every commit.
📄
GitHub Pages
Rename to index.html, push to a repo, enable Pages in Settings. Free forever.
⚠️
Remember: After deploying to a new domain, always add that domain to Firebase's Authorized Domains list and update authDomain in the HTML config.
